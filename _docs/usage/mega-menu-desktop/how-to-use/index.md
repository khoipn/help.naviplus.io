# How to use?

**Mega Menu** is one of the most popular and effective menu types, especially on desktop. Users naturally expect to find a Mega Menu right below the website header. However, Shopify‚Äôs default menu is usually simple, lacks strong structure, and offers little to no image support. That‚Äôs why you should use the Navi+ Mega Menu.

**Key Benefits:**

* Fully replaces Shopify‚Äôs simple default mega menu
* Offers multiple presentation styles with **images**, **grids**, and **multi-level** layouts for a rich, professional experience
* Works seamlessly on both mobile and desktop for a consistent experience

**FAQ:**

<details><summary><strong>1. I haven‚Äôt decided whether to use the free or paid version. Can I try the professional menu templates?</strong></summary>
<p>Absolutely! The menu groups are mainly to help beginners avoid feeling overwhelmed by advanced features in Navi+ üòÑ.\
The best way is to try out the professional menus to explore all features and layout options. Later, you can simply delete any menu you don‚Äôt need or find unsuitable.</p>
</details>

<details><summary><strong>2. Does creating a menu here make it show up on my website right away?</strong></summary>
<p>Not yet. You‚Äôll need to <strong>publish</strong> the correct menu for it to appear on your website. Don‚Äôt worry ‚Äî it‚Äôs easy!</p>
</details>

<details><summary><strong>3. What if I choose the wrong template or make a mistake?</strong></summary>
<p>No problem at all. You can delete it and create a new one anytime.\
If you're testing a menu and want to try another template, you can <strong>save</strong> the current one as a backup file and re-open it later.\
You can even <strong>copy and paste</strong> parts between menus (by opening two browser tabs) to keep your previous work. Feel free to experiment!</p>
</details>

<details><summary><strong>4. Should I create just one menu or multiple?</strong></summary>
<p>You should create <strong>multiple menus</strong>.\
Only the <strong>active</strong> menu will show on your site, but you can prepare many versions ‚Äî by selecting different templates or cloning menus.\
Menus can be configured to show/hide based on page type (home, product, collection...), device (mobile or desktop), or URL keywords.\
So go ahead and build menus that fit each situation!</p>
</details>

### Step 1: Edit the menu tree

#### 1.1. Understanding the Menu Tree

The **menu tree** is the most important part of any menu‚Äîit defines the structure.

Here‚Äôs an example of a menu tree of full desktop mega menu:

```
[+] Menu  
[+] Categories
[+] Alphabets 
Skateboarding  
   - Components (Group name)  
   - Desks
   - Trucks
   - More (Blue link to detail..)   
   - Completes      
        - Hats (Group name)
        - Burton Lunchlap Earflap Hat           
        - Volcom Ramp Stone Adjustable Hat
        - ...
[+] Accessories  
[+] Blogs  
[+] Support
[+] More
```

This is a 3-level menu tree. Level 1 includes **Menu**, **Categories, Alphabets, Skateboarding...** and **More..**.\
**Skateboarding** has 5 level-2 submenus: **Components, Desks, Completes** (Consists some level-3 menu submenus: **Hats, Burton Lunchlap Earflap Hat**...)

You‚Äôll spend most of your time building this menu tree for your website, helping customers find the content they‚Äôre looking for.&#x20;

#### **1.5. IMPORTANT!** The ‚ÄúIs Multi-Column menu item‚Äù attribute (only available for Desktop and for Mega menus).

This is a highly specialized feature designed specifically for desktop Mega Menus. It is used when you need a full-screen menu item with multiple columns. Each column can be configured with custom widths (e.g., 3/12, 4/12, 6/12...). The Multi-Column menu item is extremely powerful and flexible for building complex menu layouts tailored to your needs.

<figure><img src="https://4226540503-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F4y2gLTmg4ToH5vkorG8s%2Fuploads%2FybsBSjopbkXc3LeUUO11%2FHelp.MenuTree.1.4.3.png?alt=media&#x26;token=a04b3a4a-bbea-40f1-83d6-6dadd0b3288c" alt=""><figcaption></figcaption></figure>

If you don‚Äôt need complex layouts like Multi-Column menu items ‚Äî or plan to use them later ‚Äî you can safely delete them from your level-1 menu items.

<details><summary>See detailed instructions on how to use the ‚ÄúIs Multi-Column menu item‚Äù attribute.</summary>
<p>When a level-1 menu item is set to <strong>‚ÄúIs Multi-Column menu item‚Äù</strong>, it will be highlighted in light yellow on the Menu Tree, and its level-2 menu items will automatically be converted into <strong>Columns</strong> (with a red icon), regardless of any other attributes you assign.</p>
<p>For these <strong>Column level-2 menu items</strong>, the only usable attribute is <strong>Width layout</strong>. All sub-menu items under that column will then be arranged within a single column.</p>

<p>If you remove the <strong>‚ÄúIs Multi-Column menu‚Äù</strong> attribute from the level-1 menu item, everything will revert to the original state.</p>
<h4>Why do you need the <strong>‚ÄúDropdown width when expanded‚Äù</strong> attribute?</h4>
<p>By default, when using the <strong>‚ÄúIs Multi-Column menu item‚Äù</strong> attribute, the submenu (dropdown) expands to full screen width (or based on the container size, usually 1280, 1366, 1440, or 100% ‚Äî configurable in the <em>Advanced</em> tab).\
However, with such a wide area, you may not always have enough content or ideas to fill the space.</p>
<p>In this case, you can limit the submenu width (individually for each Multi-Column menu item) to avoid an empty or unbalanced look.</p>
<p><strong>Example:</strong> You can create a menu with a width of 700px and two columns at 6/12 each. This way, you still get a spacious layout without wasting too much screen real estate.</p>
