# なぜ私のウェブサイトにNavi+メニューが表示されないのですか？

Navi+メニューをウェブサイトに表示するには、2つの明確なオプションがあります：

1. **Embed 
[Navi+ On all page] into your theme and adjust the display in the Navi+ application**: これは最も一般的で簡単な方法です。ただし、Navi+メニューは画面に固定することができます（下部、ヘッダー、または浮遊）。調整は非常に多様で、ニーズに応じてナビゲートできるようにするための3つの異なるレベルがあります。
   * Level 1: Navi+メニューを表示/非表示にし、特定のページグループで表示/非表示にします。Shopifyでは、ホームページ、カタログ、製品、ページなどの一般的なページグループがあります。
   * Level 2: モバイルまたはデスクトップデバイスで表示/非表示にします。モバイルとデスクトップの間には全く異なる体験があるため、多くのユーザーはモバイルとデスクトップ用に別々のメニューを作成します。
   * Level 3: キーワードによる：ウェブページのURLに1つまたは複数のキーワードが含まれている場合/含まれていない場合に表示/非表示にできます。このオプションは非常に便利で柔軟性があり、各ページや各製品のために特定のメニューを作成することができます。
2. **Session 
[Navi+ On Session] is used to embed directly into each page of the Shopify theme**. このオプションでは、メニューを固定せず（下部、ヘッダー、または浮遊）、ページのコンテンツと共に流れるようにする追加の選択肢があります。この効果は、グリッドタイプのメニューやスクロールに合わせて流れるバータイプのメニューを作成したいときに便利です。

# エージェントの指示：このドキュメントを照会する

このページに直接利用できない追加情報が必要な場合は、質問をしてドキュメントを動的に照会できます。

現在のページURLに`ask`クエリパラメータを付けてHTTP GETリクエストを実行します：

```
GET https://help.shopifas.com/manual/website/help.naviplus.io/frequently-asked-questions/why-navi-not-display.md?ask=<question>
```

質問は具体的で自己完結しており、自然言語で書かれている必要があります。
応答には、質問への直接的な回答と、ドキュメントからの関連する抜粋や情報源が含まれます。

このメカニズムは、現在のページに明示的に存在しない回答が必要な場合、明確化や追加のコンテキストが必要な場合、または関連するドキュメントセクションを取得したい場合に使用します。
