# 公開フィルター — デバイス & URL ルール

公開トグルを有効にした後、2 つのフィルターレイヤーを使用してメニューが表示されるタイミングを絞り込むことができます: **デバイス** (モバイル/デスクトップ + スクロール動作) と **URL キーワード** (URL コンテンツに基づいて表示/非表示)。

---

## デバイスでフィルタリング

### プラットフォームによる有効/無効

公開モーダルの上部にある 2 つのトグル:

| 設定 | デフォルト | ノート |
|---|---|---|
| **モバイルでメニューを表示** | オン | モバイルデバイスでメニューを表示 |
| **デスクトップでメニューを表示** | オフ | デスクトップデバイスでメニューを表示 |

一部のメニュータイプは、これらのトグルのいずれかを無効にします — たとえば、デスクトップメガメニューはモバイルトグルを無効にします。

---

### スクロール動作 (スティッキーメニューのみ)

モバイルとデスクトップで別々に設定されます。

#### モバイルスクロール

| 設定 | 動作 |
|---|---|
| **下にスクロール時に自動非表示** | ユーザーが **下に** スクロールするとメニューを非表示 |
| **スクロール後に自動表示** | ユーザーが下にスクロールした後のみメニューを表示 (ページ読み込み時には表示されません) |

#### デスクトップスクロール

| 設定 | 動作 |
|---|---|
| **下にスクロール時に自動非表示** | デスクトップで下にスクロールするとメニューを非表示 |
| **スクロール後に自動表示** | デスクトップで下にスクロールした後のみ表示 |

> 自動非表示と自動表示は同時に有効にできます: メニューは下にスクロールすると非表示になり、上にスクロールすると再表示されます — ボトムタブバーの一般的なパターンです。

---

### スクリーン位置 (スティッキー / タブバーのみ)

視覚的なイラストを伴うラジオボタンセレクターから位置を選択します。

**モバイル** — 6 つのプリセット位置 (下中央、下左、下右、上のバリエーション...)

**デスクトップ** — 12 のプリセット位置 (上部バー、下部バー、左/右サイドバー、コーナー位置...)

---

### ページグループ (Shopify のみ、スティッキーメニュー)

メニューが表示される Shopify ページタイプを選択します。デフォルト: すべて。

| グループ | URL パターン |
|---|---|
| ホーム | `/` |
| 商品詳細 | `/products/*` |
| 商品カテゴリ | `/collections/*` |
| ページ | `/pages/*` |
| ブログ | `/blogs/*` |
| その他 | 残りのすべてのページ |

> グローバルマーケット (WordPress、Wix など) にはこの機能はありません。代わりに URL キーワードフィルターを使用してください。

---

## URL キーワードでフィルタリング

**"キーワードによるメニューの表示"** カード — コンテキストスライドを除くすべてのメニュータイプに適用されます。

### 2 つのフィルタータイプ

| 設定 | 動作 |
|---|---|
| **キーワードを含む** | メニューは **URL に** 少なくとも 1 つのキーワードが **含まれている** 場合にのみ表示 |
| **キーワードを含まない** | メニューは **URL に** いずれかのキーワードが **含まれている** 場合には表示されません |

複数のキーワードはカンマで区切ります `,`。

### 例

**スノーボード商品ページのみ表示:**
```
含む: snowboard
```
→ メニューは `yourdomain.com/products/the-collection-snowboard-hydrogen` のような URL でのみ表示されます。

**マルチロケーションページで非表示:**
```
含まない: multi-location
```
→ メニューは `yourdomain.com/products/the-multi-location-snowboard` では **表示されません**。

**組み合わせ:**
```
含む: collections
含まない: sale, outlet
```
→ コレクションページでのみ表示されますが、"sale" と "outlet" コレクションでは非表示です。

### ノート

- フィルターは **完全な URL 文字列の一致** を使用します。正規表現ではありません。
- キーワードは **大文字と小文字を区別しません**。
- 空のままにすると = フィルターなし (メニューはすべてのページに表示されます)。
- 両方のフィルターは AND で結合されます — 両方の条件を満たす必要があります。
