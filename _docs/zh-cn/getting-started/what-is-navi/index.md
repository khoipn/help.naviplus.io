# 什么是 Navi+？

Navi+ 是一个 **无代码导航构建器** — 它让您可以为任何网站创建美观、高转化率的菜单，而无需触及您的主题或编写一行代码。

如果您网站的访客在寻找他们想要的内容时遇到困难，您的菜单可能让他们失望。Navi+ 可以帮助您在几分钟内解决这个问题。

## 五种菜单类型

选择适合您用例的菜单：

| 菜单类型 | 最适合 |
| --- | --- |
| **Tab Bar** | 移动优先导航，感觉像本地应用 |
| **Mega Menu** | 带有图像、列和链接的丰富下拉菜单 — 具有单独的桌面和移动版本 |
| **Slide Menu** | 具有多级支持的汉堡菜单 |
| **Grid Menu** | 视觉类别浏览 — 适合电子商务 |
| **FAB** | 快速访问的浮动按钮，始终触手可及 |

您可以在同一网站上同时使用多种菜单类型。

## 它是如何工作的

Navi+ 是一个 **小部件** — 它不会替换或修改您现有的主题。相反，它会将一个轻量级脚本注入到您的页面中，在您现有的布局上渲染菜单。

您可以从 **Navi+ 仪表板** 构建和管理所有内容。更改会立即生效 — 无需编辑代码、修改模板或重新发布您的主题。

以下是流程：

1. 您在仪表板中创建一个菜单
2. Navi+ 生成一个小脚本
3. 该脚本嵌入到您的网站中（通过 Shopify 的 App Block，或其他地方的代码片段）
4. 菜单在每次页面加载时在您现有的布局上渲染

## 与您的平台兼容

Navi+ 适合您已经使用的工具：

**Shopify** · **WordPress** · **WooCommerce** · **Wix** · **Webflow** · **Squarespace** · 以及任何接受 `<script>` 标签的网站。

## 什么是 Navi+ 不是

- **不是一个主题** — 它与您现有的主题并行工作，而不是替代它
- **不是您手动安装的代码** — 一个 App Block（Shopify）或一个嵌入代码片段就足够了
- **不是可能会破坏您网站的插件** — 如果 Navi+ 被禁用，您的原始主题菜单会自动恢复

## 为速度而生

Navi+ 资产通过 Cloudflare CDN 以 ~52ms 的平均加载时间交付 — 通常比您平台自己的脚本更快。在第一次加载后，菜单会被缓存并立即加载。
